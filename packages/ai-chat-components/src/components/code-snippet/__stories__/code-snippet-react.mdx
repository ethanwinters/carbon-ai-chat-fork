## Overview

The `<CodeSnippet>` and `<CodeSnippetCard>` components wrap the `cds-aichat-code-snippet` web component to provide syntax highlighting, streaming updates, and editable code blocks inside React applications.

### Getting started

Here's a quick example to get you started using the `code` prop (recommended):

```jsx
import CodeSnippet from "@carbon/ai-chat-components/es/react/code-snippet.js";

function ReadOnlySnippet({ code }) {
  return (
    <CodeSnippet highlight code={code} />
  );
}
```

### Using children (alternative)

You can also provide code via children, though the `code` prop is recommended for better performance, especially when streaming content:

```jsx
import CodeSnippet from "@carbon/ai-chat-components/es/react/code-snippet.js";

function ReadOnlySnippet({ code }) {
  return (
    <CodeSnippet highlight>
      {code}
    </CodeSnippet>
  );
}
```

**Performance Note:** The `code` prop is more efficient than children because it bypasses the slot change detection mechanism. This is particularly beneficial when streaming code content, as it reduces unnecessary DOM operations and improves rendering performance.

### Usage Patterns

#### Streaming With AI Responses (Recommended)

```jsx
function StreamingSnippet({ stream }) {
  const [content, setContent] = React.useState("");

  React.useEffect(() => {
    const subscription = stream.subscribe((chunk) =>
      setContent((previous) => previous + chunk),
    );
    return () => subscription.unsubscribe();
  }, [stream]);

  return (
    <CodeSnippet highlight code={content} />
  );
}
```

#### Editable Playground

```jsx
function EditableSnippet() {
  const [content, setContent] = React.useState("// Start typing...");

  return (
    <CodeSnippet
      editable
      highlight
      code={content}
      onChange={(event) => {
        setContent(event.detail.value);
      }}
    />
  );
}
```
