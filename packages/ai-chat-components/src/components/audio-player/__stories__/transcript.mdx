# Transcript

`<cds-aichat-transcript>` provides an accessible, expandable/collapsible interface for displaying audio and video transcripts.

## Key Features

- **Expandable/collapsible**: Clean interface that expands on demand
- **Markdown support**: Render formatted transcript text
- **Accessibility**: Proper ARIA attributes and keyboard navigation
- **Event handling**: Custom events for expand/collapse state changes
- **Internationalization**: Support for multiple languages

## Use Cases

The transcript component is ideal for:

- Providing text alternatives for audio content
- Meeting accessibility requirements (WCAG)
- Allowing users to search or reference specific content
- Supporting users who prefer reading over listening

## Getting started

Here's a quick example to get you started.

```html
<cds-aichat-transcript
  text="My text input is, you know, I am a teapot and then my image input is a picture of David Hasselhoff."
  label="English Transcript"
  language="en"
>
</cds-aichat-transcript>
```

### With Audio Player

The transcript component works seamlessly with the audio player in a Card layout:

```html
<cds-aichat-card>
  <div slot="media" data-rounded="top">
    <cds-aichat-audio-player
      source="https://example.com/audio.mp3"
      aria-label="Audio title"
    >
    </cds-aichat-audio-player>
  </div>
  <div slot="body">
    <h4>Audio Title</h4>
    <p>Audio description</p>
    <cds-aichat-transcript
      text="Transcript text here..."
      label="English Transcript"
      language="en"
    >
    </cds-aichat-transcript>
  </div>
</cds-aichat-card>
```

## Events

The component fires the following custom event:

```javascript
const transcript = document.querySelector('cds-aichat-transcript');

transcript.addEventListener('cds-aichat-transcript-toggle', (event) => {
  console.log('Transcript expanded:', event.detail.expanded);
});
```
